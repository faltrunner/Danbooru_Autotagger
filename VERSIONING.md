# 版本控制政策 (Semantic Versioning)

本專案採用 **語義化版本 (Semantic Versioning 2.0.0)** 和**雙版本軌制**。

## 版本格式

```
腳本版本：MAJOR.MINOR.PATCH
字典版本：MAJOR.MINOR.PATCH
```

---

## 腳本版本 (`danbooru-ai-tagger.user.js`)

### MAJOR 版本遞增 (X.y.z → (X+1).0.0)

**何時遞增**：
- 架構重構（如 IIFE → Module 轉換）
- 不兼容的 UI 改動（變更選擇器、快捷鍵等）
- 不兼容的 API 改動
- Tampermonkey API 層級變更

**範例**：
- `0.x.x` → `1.0.0`：從本地 `@resource` 改為 CDN 字典（架構級改動）

---

### MINOR 版本遞增 (x.Y.z → x.(Y+1).0)

**何時遞增**：
- ✅ 新功能新增（AI 標籤預測、搜尋功能、右鍵上傳等）
- ✅ 支援新的網站/平台
- ✅ UI 增強（不破壞現有 UI）
- ✅ 效能改進（顯著提升）

**不遞增 PATCH**（重置為 0）：因為新功能可能同時帶來小 bug 修復，PATCH 應從 0 重新計數。

**範例**：
- `1.0.0` → `1.1.0`：新增來源 URL 偵測功能
- `1.1.0` → `1.2.0`：新增全網右鍵上傳
- `1.2.0` → `1.3.0`：新增 DOM 智慧提取邏輯

---

### PATCH 版本遞增 (x.y.Z → x.y.(Z+1))

**何時遞增**：
- 🐛 **Bug 修復**（不新增功能）
  - 修正選擇器失效
  - 修正渲染競態條件
  - 修正 API 呼叫失敗
  - 修正字典載入異常
- 🔧 **向後相容的小優化**
  - 代碼效率改進
  - UI 細微調整（邊距、顏色）
  - 註釋與文檔更新
- 📱 **安全性補丁**（XSS、注入等）

**不應包含**：
- ❌ 新功能
- ❌ UI 大幅改動
- ❌ 選擇器變更

**範例**：
```
v1.3.0 → v1.3.1：修正 Pixiv /artworks/ 正規表達式失配
v1.3.1 → v1.3.2：修正 Twitter DOM 選擇器在某些頁面失效
v1.3.2 → v1.3.3：改進 URL 修正效能，減少正規表達式編譯次數
```

---

## 字典版本 (`DICTIONARY.json`)

### MAJOR 版本遞增 (X.y.z → (X+1).0.0)

**何時遞增**：
- 字典格式根本改變（如 `{"tag": "中文"}` 改為 `{"tag": {"zh": "中文", "ja": "日本語"}}`）
- 編碼方式改變（UTF-8 → 其他）
- 舊版本無法向後相容新字典

---

### MINOR 版本遞增 (x.Y.z → x.(Y+1).0)

**何時遞增**：
- 新增 10+ 條新標籤
- 添加新的分類（如全新遊戲、新動畫等）
- 大規模譯名更正（50+ 條）
- 新增字典欄位（同時保持向後相容）

**範例**：
```
v1.0.0 → v1.1.0：新增 Vtuber 相關標籤 50 條
v1.1.0 → v1.2.0：新增遊戲分類 30+ 標籤
```

---

### PATCH 版本遞增 (x.y.Z → x.y.(Z+1))

**何時遞增**：
- 修正單個/少數標籤的譯名（1~5 條）
- 修正標籤拼寫（如 `koisnow` → `konosuba`）
- 移除重複標籤
- 添加缺失的漢字標音

**範例**：
```
v1.0.0 → v1.0.1：修正「原神」標籤譯名
v1.0.1 → v1.0.2：移除 5 個重複標籤
```

## 更新觸發機制

### 自動更新檢查

```javascript
// @metadata
SCRIPT_VER = "1.3.0"
DICT_VER = "1.0.0"
```

使用者端：
1. 每次打開 Danbooru 時檢查版本
2. 若腳本版本更新 → 提示更新（或自動更新）
3. 若字典版本更新 → 清除快取，重新下載

## 版本發布檢查清單

### 發布新版本前：

- [ ] 版本號已在 `@version` 更新
- [ ] PROGRESS.md 已記錄新版本條目
- [ ] README.md 已更新版本表格
- [ ] 功能測試無誤（至少在 Chrome + Firefox 測試）
- [ ] 字典 JSON 格式驗證無誤（若有更新）
- [ ] Git commit 訊息清晰（格式：`feat:` / `fix:` / `chore:` 等）
- [ ] 無明顯 console errors

### 發布步驟：

```bash
# 1. 更新版本號
# → @version 在 .user.js 頂部
# → PROGRESS.md 新增版本條目
# → README.md 更新版本表格

# 2. 提交 Git
git add -A
git commit -m "feat: v1.x.0 description"
# 或
git commit -m "fix: v1.0.x - bug fix description"

# 3. 推送
git push origin master
```

## 常見問題

**Q: 為什麼同時有腳本版本和字典版本？**

A: 因為：
- 腳本更新頻率：功能改進、bug 修復（通常 ~1-2 週）
- 字典更新頻率：標籤新增、譯名修正（通常 ~1 個月）

獨立版本號能精確追蹤各部分的演進。

---

**Q: 什麼時候該遞增 MAJOR？**

A: 當使用者的「現有設定/使用習慣」會破裂時：
- UI 選擇器改變 → 現有 CSS override 失效
- 快捷鍵改動 → 肌肉記憶失效
- 字典格式改變 → 舊版腳本無法讀取

---

**Q: 我發現了小 bug，應該提 v1.3.1 還是 v2.0.0？**

A: **查看是否破壞兼容性**：
- 只修復 bug，未新增/移除功能 → **PATCH（v1.3.1）** ✓
- 改動了 UI 選擇器或快捷鍵 → **MAJOR（v2.0.0）** ⚠️

---

**Q: 為什麼字典版本一直是 1.0.0？**

A: 字典尚未有量級變更（如格式改變），故保持 1.0.0。當：
- 新增 Vtuber 分類 50+ 標籤 → `1.1.0`
- 大規模譯名修正 → `1.x.0`
- 格式改變（如新增日文譯名） → `2.0.0`

```
v0.1.0  - 初版核心功能（AI 標籤）

v1.0.0  - CDN 字典架構（架構重構 → MAJOR）

v1.1.0  - 來源 URL 偵測（新功能 → MINOR）
v1.2.0  - 全網右鍵上傳（新功能 → MINOR）
v1.2.1  - 修正 Twitter DOM 選擇器（bug fix → PATCH）
v1.3.0  - 智慧 URL 提取邏輯（新功能 → MINOR）
v1.3.1  - 修正 Pixiv regex（bug fix → PATCH）
v2.0.0  - [未來] 從 script 改為 extension（架構 → MAJOR）
```
